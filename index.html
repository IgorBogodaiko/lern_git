до папки git_test я підключу git, для цього меті потрібно в даній папці відкрити консоль гітбаш та ввести команду git init

git init - ініціалізуєм гіт, грубо кажучи, поселяємо його в папку з проектом. щоб подивитися які файли попали в гіт потрібно ввести команду git status

git add . добавить всі файли в область StageArea з якої гіт почне слідкувати за змінами та файлами

git commit -m 'кометн який ми лишаємо' добавляє файли для постійного збереження, а також коментар

git log подивития коміти та коменти

git commit -a -m 'comment text' команда яка за один раз добавляє зміни що сталиса в файлі після редагування а також робить комміт та комент до нього, по суті це скорочений запис команд адд та комміт

далі іду гітхаб вибираю нюв репозеторі, даю назву, я назвав lern_git жму кнопку креате, мене перекидає створення або копіювання з локального диски я вибираю копіювання для цього виділяю команду git remote add origin https://github.com/IgorBogodaiko/lern_git.git і вставляю в консоль. Потім копіюю команду git push -u origin master яка означає що я вигружаю папку з проектом в створений репозеторій, далі коли команда виповнилася обновляю гітхаб

тепер як сайт вігружений на гітхаб, скачаю його під видом іншого розробника та з допомогою команди комадди branch створю парелальну копию в якій буду дописувати дане об яснення, після цоьго з здпомогою команди merge об єднаю гілки та закомітю

git branch branch2 створив гілку branch2, командою git checkout branch2 перейшов в неї, зараз робитиму комміт з коментом fist commit on branch2

зараз зроблю ще один комміт, перейду на головну гілку master зроблю там комміт.
